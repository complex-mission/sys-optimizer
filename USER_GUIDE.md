## 快速上手

1. 打开应用,首次使用需同意《使用条款》。
2. 停在"智能扫描"页,直接点**开始扫描**(默认"标准"挡)。
3. 扫描完成后,列表按大小排序。**蓝色的项**默认已勾选,可放心清理;琥珀色和灰色的项请看说明再决定。
4. 点**清理选中的 X GB**,确认后完成。

> 应用启动时会请求管理员权限(UAC 弹窗)。这是为了能清理系统临时文件、更新缓存等需要权限的位置。

## 核心概念:风险三色

Cache Insight 给每一项待清理内容标了颜色,这是整个应用最重要的约定:

| 颜色 | 含义 | 默认状态 | 说明 |
|---|---|---|---|
| 🔵 **蓝色** | 无风险缓存 | **默认勾选** | 删了系统或软件会自动重建,你几乎无感 |
| 🟠 **琥珀色** | 重建耗时 | **默认不勾** | 删了不丢数据,但重新生成可能很慢(如视频代理要重跑几小时),或需重新下载 |
| ⚪ **灰色** | 仅报告 | **无法删除** | 这是你的内容(如会议录像),Cache Insight **只显示体积,不提供删除按钮** |

启动时顶部的提示横幅就是在讲这三色。看懂它,就看懂了整个应用。

## 谨慎删除模式

在**设置**页有一个"谨慎删除模式"开关(**默认开启**),它决定高风险项删除时的去向:

| 项目类型 | 谨慎模式开启(默认) | 谨慎模式关闭 |
|---|---|---|
| 🔵 无风险缓存 | 永久删除(立即释放空间) | 永久删除 |
| 🟠 重建耗时项 | **移入回收站**(可反悔) | 永久删除(立即释放空间) |
| ⚪ 仅报告 | 从不删除 | 从不删除 |

- **为什么缓存类始终永久删除**:蓝色项删了会自动重建,移进回收站只会白占空间、还得手动清空,失去"清理"的意义。
- **为什么琥珀色项默认进回收站**:这些重建代价高(如视频代理要重跑数小时、依赖要重新下载),给你一个反悔的机会更稳妥。
- **注意**:移入回收站的项目,**磁盘空间要到你清空回收站后才真正释放**。如果你就是要立刻腾空间,可以在设置里关闭此模式,让琥珀色项也永久删除。

## 1. 智能扫描

一键清理最常见的缓存和临时文件。提供三挡,由你决定扫多深:

| 挡位 | 耗时 | 扫描内容 |
|---|---|---|
| **快速** | 几秒 | 临时文件、浏览器缓存、错误报告、回收站 |
| **标准**(推荐) | 约 1 分钟 | 快速的全部 + 缩略图缓存、更新缓存、已安装专业软件的缓存 |
| **深度** | 几分钟 | 标准的全部 + 更彻底的项(如包管理器/构建缓存:npm、Yarn、pnpm、Gradle、Flutter pub 等) |

### 扫得到

**快速挡就能清理的(即时见效):**

- **用户临时文件**(`%TEMP%`)—— 各种软件产生的临时文件
- **系统临时文件**(`C:\Windows\Temp`)—— 需管理员权限
- **Chrome / Edge / Firefox 浏览器缓存** —— 各账户(Profile)的网页缓存
- **Windows 错误报告与崩溃转储**
- **回收站** —— 清空所有磁盘的回收站

**标准挡追加:**

- **缩略图 / 图标缓存** —— 资源管理器的预览图缓存,删除后自动重建
- **Windows 更新缓存** —— 已下载的更新安装包,需管理员权限
- **各类专业软件缓存** —— 见[软件专项清理](#2-软件专项清理)

**深度挡追加:**

- **Prefetch 预读取文件**(`C:\Windows\Prefetch`)—— 程序启动优化文件,可安全清理,系统会自动重建(需管理员权限)
- **Windows 日志 / 内存转储**(`C:\Windows\Logs`、`C:\Windows\Minidump`)—— 诊断日志与蓝屏 Minidump(需管理员权限)
- **包管理器与构建缓存** —— npm / Yarn / pnpm / Gradle / Flutter pub / Cargo / Go / Maven / NuGet / pip / Conda 等,见[软件专项清理](#2-软件专项清理)

### 扫不到 / 不碰

- ❌ **浏览器的书签、密码、历史记录、Cookie 登录状态** —— 只清缓存,绝不动这些。清理后你不会被登出、书签不会丢。
- ❌ **Arc、Opera 等其他浏览器** —— 目前内置 Chrome / Edge / Firefox 的规则(其他浏览器的缓存可通过"空间分析"手动查看)。
- ❌ **正在被占用的文件** —— 会自动跳过,清理结果里会告诉你跳过了几个。
- ❌ **快捷方式指向的真实文件、符号链接的目标** —— 不跟随链接,不会误删链接指向的内容。

### 清理后会怎样

这些都是缓存,删除后**系统和软件会按需自动重建**。唯一可感知的变化:

- 首次打开资源管理器某些文件夹,缩略图会重新生成(略慢一下);
- 浏览器首次访问网站会稍慢(缓存重建);
- 回收站清空后**无法恢复**(界面对回收站有明确标注)。

### 几个重要说明

- **达芬奇代理文件(🟠)**:删除不丢素材,但重新生成代理可能要**几小时**,所以默认不勾选。如果你已经把代理路径改到了别的盘,扫描时可能显示 0 —— 这时点"指定路径"选中你的实际代理目录即可。
- **Spotify 离线缓存(🟠)**:这里面是你**下载的离线歌曲**,删了离线歌单会失效、需要重新下载,所以默认不勾。
- **IDE 缓存(Android Studio / JetBrains,🔵)**:只清理各产品版本目录下的 `caches`、`index`、`log`、`tmp` 四个缓存子目录,IDE 会自动重建(下次打开首个项目重建索引会略慢一次)。**绝不触碰**你的 IDE 设置、插件,也**绝不触碰 JetBrains Toolbox 里安装的 IDE 本体**。
- **包管理器缓存与构建缓存(Gradle / Flutter pub / pnpm / Cargo / Go / Maven / NuGet,🟠)**:这些不是垃圾,而是已下载的依赖包 / 构建产物。删了不丢代码,但下次构建会**重新下载或重新编译**,可能较慢或耗流量,所以默认不勾。npm / Yarn / pip / Composer 等纯下载缓存重建代价较小,按无风险处理。
- **Go 模块缓存的特别说明**:Go 把模块缓存里的文件设为**只读**,永久删除时会大量跳过。建议开启[谨慎删除模式](#谨慎删除模式)(移入回收站可绕过只读),或直接用命令 `go clean -modcache`。
- **通讯软件缓存(Teams / Slack / Discord,🔵)**:这些是 Electron 应用,清理只碰它们的 `Cache / Code Cache / GPUCache` 等网页缓存目录,**不含聊天记录、消息和你收发的文件**。
- **Docker / Zoom(⚪)**:Docker 数据关系到你的镜像和容器,Zoom 录制文件是你的会议录像 —— 这些**只报告体积,不提供删除**,需要你自己判断处理。
- **微信 / QQ(⚪ 仅报告)**:微信、QQ 把**你接收的图片、视频、文件**和缓存**混在同一个目录**里(常达数十 GB),难以自动、安全地只挑出缓存。为避免误删你的聊天文件,Cache Insight 对它们**只报告体积、提供"打开位置",不自动删除** —— 你可以打开目录自行清理(如微信的 `设置 → 存储空间`)。

### 扫不到

- ❌ **规则库里没有的软件** —— 只有列出的软件会被专项识别。其他软件的缓存,可以用"空间分析"手动找到。
- ❌ **项目级的自定义路径** —— 达芬奇等软件可以为每个项目单独设代理路径,这种项目内的自定义位置目前无法自动发现,需手动指定。

## 3. 空间分析

回答那个最让人困惑的问题:**"我什么都没干,磁盘怎么就满了?"**

选一个磁盘,Cache Insight 会像 TreeSize 那样,把空间占用**逐层画出来**——用色块大小直观表示谁占得多,点击色块可以一层层下钻进去,直到找到真正的"大户"。

### 用得到的场景

- 磁盘莫名其妙满了,想知道到底是什么占的;
- 怀疑某个软件/游戏/项目文件夹特别大,想确认;
- 找到大目录后,可以"打开位置"直接去处理。

### 特点

- **两种视图**:方块图(treemap,直观看比例)和列表(精确看数值),随时切换;
- **边扫边看**:大磁盘扫描需要时间,过程中会实时显示已扫描的大小,不用干等;
- **纯查看,不删任何东西** —— 这是个"看清"工具,不是清理工具。找到大目录后,该清理的引导你去清理,属于你的数据只告诉你在哪。

### 值得关注的位置(不建议自动清理)

选盘界面下方会列出一批**"见得到、但不该自动删"的已知位置**,点"分析"可一键跳过去看它们占了多大,点"打开位置"直接去处理:

| 位置 | 为什么不自动删 |
|---|---|
| **Windows Installer**(`C:\Windows\Installer`) | 已安装程序的安装/修复源,删了会破坏卸载、修复、更新 |
| **Microsoft Office 安装缓存**(ClickToRun) | Office 安装/更新源,删了会导致 Office 无法卸载/修复 |
| **字体缓存** | 被字体服务独占,需停服务才能安全重建 |
| **Windows 搜索索引** | 被搜索服务独占,应在"索引选项"里重建而非直接删 |
| **传递优化缓存**(更新 P2P) | 用系统自带"磁盘清理"清除更稳妥 |
| **Maya 用户数据** | 工程、预设与缓存混在一起,需你自己区分 |

> 只列出**实际存在**的位置。这一区块延续空间分析"只看清、不代删"的定位——把风险高或系统占用的东西摆出来让你知道,怎么处理由你决定。

### 扫不到 / 看不到

- ❌ **不跟随符号链接** —— 避免把链接目标的体积重复计算。
- ⚠️ **需要权限的目录**(如其他用户的文件夹、部分系统目录)可能读不到,会被跳过,不计入统计。因此显示的总量可能略小于磁盘实际占用。

## 4. 大文件

在指定目录里找出**占地方的大文件**,按大小排序。

### 怎么用

1. 选择要扫描的文件夹(默认你的用户目录,可改成任意文件夹或整个磁盘);
2. 选阈值:**50MB / 100MB / 500MB / 1GB**(只显示大于此值的文件);
3. 扫描,结果按大小从大到小排列;
4. 勾选想删的,**移入回收站**(可恢复)。

### 扫得到

- 你选定目录下(含所有子文件夹)大于阈值的**所有文件**。

### 扫不到 / 不碰

- ❌ **系统目录**:`C:\Windows`、`Program Files`、`Program Files (x86)`、`ProgramData`、回收站、System Volume Information —— 这些**不会被扫描**。原因有二:一是扫它们又慢又没意义,二是防止你误删系统文件。
- ❌ **小于阈值的文件** —— 按你选的阈值过滤。

### 安全说明

- 大文件是**你自己的文件**,不是垃圾。所以删除**一律移入回收站**,后悔了能从回收站找回来。
- 即使你手动指定了系统目录里的文件,程序也会拒绝删除(双重保护)。

## 5. 重复文件

找出内容**完全相同**的文件(不是文件名相同,是内容逐字节相同),帮你删掉多余的副本。

### 怎么判断"相同"

Cache Insight 用三步逐级筛查,既准又快:

1. **先比大小** —— 大小不同的文件绝不可能相同,先排除掉绝大多数;
2. **再比首尾采样** —— 大小相同的,快速比对文件头尾各 64KB;
3. **最后完整校验** —— 前两步都一样的,才做完整内容比对确认。

这样保证**只有内容真正逐字节相同的文件才会被判为重复**,不会误判。(比如两个大小相同、开头结尾也一样、只有中间不同的文件,会在第三步被正确区分开。)

### 怎么用

1. 选择要扫描的文件夹;
2. 扫描后,重复文件按组显示,每组是一批内容相同的文件;
3. **每组默认自动保留一个**,其余标记为删除 —— 你可以改选保留哪一个;
4. 确认后,标记删除的文件**移入回收站**。

### 关键安全设计

- ⭐ **每组至少保留一个,无法全删** —— 界面会阻止你把一组文件的所有副本都删掉,从根本上防止"把某个文件的所有备份一次删光"的事故。
- 删除**移入回收站**,可恢复。

### 扫不到 / 不碰

- ❌ **小于 4KB 的小文件** —— 这类文件数量极多但重复价值低,忽略以提升速度。
- ❌ **系统目录** —— 同大文件模块,不扫 Windows / Program Files 等。
- ❌ **不跟随符号链接**。

## 6. 启动项

管理**开机自动启动**的程序,给系统提速。

### 看得到

从三个来源汇总所有开机自启项:

- **注册表 Run 键**(当前用户 + 所有用户)—— 最常见的自启方式;
- **启动文件夹** —— 放在里面的快捷方式;
- **计划任务** —— 设置为"登录时/启动时"触发的任务。

每项显示:名称、启动命令/路径、来源、是否"所有用户"级别、当前是否启用。

### 操作方式:只禁用,不删除

- ⭐ **关掉开关 = 禁用,不是删除**。这和 Windows 任务管理器的做法完全一致 —— 禁用只是让它开机不启动,原始项完整保留,**随时可以重新打开**。
- 因此这个模块**很安全**:即使禁错了,重新打开开关就恢复了。

### 注意

- ⚠️ **"所有用户"级别的启动项**(标了标签的)修改需要管理员权限。如果关闭失败,界面会提示需要管理员权限 —— 应用本身已以管理员运行,一般不会遇到。
- 不确定某项是什么?建议先查一下再禁用。禁用没有风险(可恢复),但禁错了可能导致某些软件不再随开机启动。

## 7. 卸载残留

查找软件卸载后**可能残留**的文件夹。

### ⚠️ 请先理解:这是"猜测",且只报告

这个模块的原理是:读取系统里"已安装软件"的列表,再去 `Program Files`、`AppData` 等位置找那些**对不上任何已安装记录的文件夹**,把它们当作疑似残留列出来。

这种判断天生是**启发式的猜测**,可能出错:

- 可能把你**还在用的绿色软件**(免安装软件)当成残留;
- 也可能漏掉一些真的残留。

正因如此,Cache Insight 对这个模块的处理是:

- ⭐ **只列出报告,完全不提供删除功能** —— 界面上连删除按钮都没有;
- 每项标了**参考置信度**("较可能" / "存疑"),但仅供参考;
- 你只能"打开位置"去自己查看、自己判断是否手动删除。

### 看得到

- `Program Files` / `Program Files (x86)`(要求文件夹里有 .exe/.dll,降低误报)
- `AppData` / `LocalAppData`

只列出大于 64KB、且近 90 天内没被修改过的目录(近期动过的更可能是活跃软件,会降级或排除)。

### 看不到 / 排除

- ❌ **系统公共目录**(Common Files、Windows、Microsoft 等)—— 白名单排除,不会误报。
- ❌ **近期修改过的目录** —— 视为可能活跃,降级为"存疑"或不报。

**一句话:这个模块只帮你发现线索,删不删、怎么删,完全由你决定。**

## 8. 系统级空间

⚠️ **本模块风险最高,请务必读完说明再操作。** 这里能回收系统占用的**大块空间**(动辄十几 GB),但每一项都有代价,而且**不像回收站那样容易撤销**。所以默认全部关闭,需要你逐项确认。

| 项目 | 能省多少 | 代价 |
|---|---|---|
| **休眠文件** | 通常是内存大小的 40–75%(如 16G 内存约 6–12GB) | 关闭后**无法使用休眠和快速启动** |
| **系统还原点** | 数 GB 到数十 GB | 收缩存储上限,**较旧的还原点会被清除**,能回滚的时间点变少 |
| **Windows.old** | 10–30GB | 上次系统升级的备份,清理后**无法回退到旧版本** |
| **WinSxS 组件存储** | 可回收数 GB | 清理被取代的旧组件,清理后**无法卸载已安装的更新** |

### 每一项的建议

- **休眠文件**:如果你从不用"休眠"功能(注意不是"睡眠"),关掉能立刻省下一大块。但关掉后"快速启动"也会失效(开机可能略慢几秒)。
- **系统还原点**:如果你依赖系统还原来应对故障,谨慎操作。这项是收缩上限到 1GB,不是全删,但旧还原点会没。
- **Windows.old**:升级 Windows 后如果确认新系统一切正常、不打算回退,可以清理,能省很多。
- **WinSxS**:清理相对安全,主要影响是不能再卸载旧更新。分析这一项可能需要几十秒(系统在盘算能清多少)。

### 说明

- 所有操作调用 Windows 官方工具(powercfg / DISM / vssadmin)完成,不是暴力删文件。
- 每项操作前都会**再次弹窗确认**,并把代价再说一遍。
- 已经处于目标状态的项(如休眠已关闭)会显示为不可操作。

## 9. 系统信息

查看这台电脑的**硬件配置**,纯展示,不改任何东西。

### 看得到

- **CPU**:型号、核心数、线程数、频率;
- **主板 / BIOS**:厂商、型号、BIOS 版本;
- **显卡**:型号、显存(参考值);
- ⭐ **内存插槽**:这是重点 —— 主板**总共几个插槽、用了几个、还剩几个空的**,每根内存的容量、频率、代际(DDR4/DDR5)、品牌、型号。想加内存时,一眼就知道还有没有空位、现有内存什么规格;
- **存储**:每块硬盘的型号、容量;
- **显示器**:分辨率;
- **电池**(笔记本):健康度(当前满充容量 / 出厂设计容量)。

还可以**一键复制整份报告**,发给 IT 或朋友帮忙看配置时很方便。

### 看不到(需要专门硬件驱动,本工具不做)

- ❌ **CPU / 显卡温度**
- ❌ **风扇转速**
- ❌ **电压**
- ❌ **内存的 XMP 档位、时序(CL 等详细参数)**

这些数据必须加载专门的内核级驱动才能读取(CPU-Z、AIDA64 就是这么做的)。加载内核驱动有安全和稳定性风险,还容易被杀软报警,对一个日常工具来说不值得,所以 Cache Insight **有意不做**。想看温度请用专门的硬件监控软件。

### 关于数据准确性

硬件信息来自主板固件(SMBIOS)。部分厂商填写的信息不规范,可能出现某些字段显示为"厂商未提供" —— 这是厂商没填,不是软件的问题。

## 常见问题

**Q:为什么打开就要管理员权限?**
A:清理系统临时文件、Windows 更新缓存,以及回收系统级空间,都需要管理员权限。为了功能完整,应用启动即请求提权。

**Q:清理后能恢复吗?**
A:分情况——
- 缓存类(蓝色):删了会自动重建,不需要"恢复";
- 大文件、重复文件:移入回收站,可从回收站恢复;
- 回收站、系统级空间(休眠/还原点/Windows.old):**不可恢复**,界面对这些都有明确提示。

**Q:会上传我的数据吗?**
A:不会。Cache Insight 不收集、不上传任何数据。所有扫描和清理都在你本机完成,操作日志也只存在本机(可在设置里打开日志目录查看)。

**Q:清理会不会把我的文件/文档删掉?**
A:不会。智能扫描和软件专项只清理明确的缓存/临时目录,绝不进入你的文档、照片、项目文件。此外清理引擎内置**危险目录安全闸**:即便某条规则或手动设置的路径异常地指向了盘符根(如 `C:\`)、系统目录根(Windows / Program Files / ProgramData)、你的个人文件夹(桌面 / 文档 / 下载等)或 AppData 根,引擎都会**拒绝对其递归清理**。大文件和重复文件虽然扫你的目录,但删除都进回收站可恢复,而且要你自己勾选。

**Q:为什么有些项显示 0,但我知道那软件占了很多空间?**
A:很可能你把该软件的缓存路径改到了别的位置。在"软件专项"页点该项的"指定路径",选中你的实际目录即可。为防误操作,**如果你指定的是系统目录或盘符根等危险位置,程序会拒绝并给出提示** —— 只需选择真正的缓存目录即可。

**Q:第一次运行提示"Windows 已保护你的电脑"?**
A:这是因为本工具没有购买代码签名证书(内部工具),不是病毒。点"更多信息 → 仍要运行"即可。如有顾虑,可核对官网提供的文件 SHA-256 校验值。

**Q:支持 Windows 7 吗?**
A:不支持。仅支持 Windows 10(1809 及以上)和 Windows 11,64 位。

**Q:界面能切英文吗?**
A:能。设置页可切换中文 / English / 跟随系统。

**Q:为什么不清 `C:\Windows\Installer`?听说能省好几个 GB。**
A:这是**危险的常见误区**。`C:\Windows\Installer` 里的 `.msi/.msp` 是已安装程序的安装/修复源,删了会导致相关软件**无法卸载、修复或更新**,甚至报错。这里的文件看似冗余,实则被 Windows 按需引用,**Cache Insight 有意不碰**。真要瘦身应使用微软的 `PatchCleaner` 等专门工具谨慎处理。

## Cache Insight 不做的事

为了让你放心,这里明确列出本工具**有意不做**的事:

- ❌ **不删除你的个人文件**(文档、照片、视频、项目等),缓存清理绝不触碰这些区域;
- ❌ **不动浏览器书签、密码、历史、登录状态**;
- ❌ **不删除会议录像、离线音乐、Docker 数据、微信/QQ 接收的文件**等你的内容(这些只报告);
- ❌ **不清 Windows Installer 目录**(`C:\Windows\Installer`)—— 删了会破坏程序的卸载/修复/更新,是危险误区;
- ❌ **不动字体缓存、Windows 搜索索引**等被系统服务独占的项 —— 需停服务才能删,强删易致服务异常;这些已列入[空间分析的"值得关注的位置"](#3-空间分析)供你查看体积、自行处理;
- ❌ **不自动处理 Lightroom 预览、C4D/Maya 工程缓存** —— 它们与你的目录/工程混在一起,难以安全区分;Maya 用户数据同样列入空间分析的"值得关注的位置",其余可用空间分析手动查看;
- ❌ **不做注册表"优化""清理"** —— 收益极小、风险不低,业界已普遍不建议;
- ❌ **不加载内核驱动**,因此不读温度/风扇/电压;
- ❌ **不联网上传任何数据**;
- ❌ **卸载残留不自动删除**,只报告;
- ❌ **启动项不删除**,只禁用(可恢复)。

> 注:**Windows.old、休眠文件、系统还原点、WinSxS 组件存储**等系统级大块空间,在[系统级空间](#8-系统级空间)模块中用微软官方工具(DISM / powercfg / vssadmin)安全回收,不在此列。**传递优化(Windows Update P2P)缓存**已列入[空间分析的"值得关注的位置"](#3-空间分析),可查看体积后用系统自带的"磁盘清理"处理。
